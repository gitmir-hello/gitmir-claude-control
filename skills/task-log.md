Веди журнал выполненных задач этого проекта в файле `.claude/tasks.json`
(относительно корня текущей рабочей директории). Его читает локальный дашборд
Claude Control и показывает, что было сделано в проекте.

Сделай сразу:
1. Прочитай `.claude/tasks.json`, если он есть. Если нет — создай папку `.claude/`
   (если нужно) и файл с пустым журналом по схеме ниже. Существующие записи не затирай.
2. Коротко подтверди: «Веду журнал задач в `.claude/tasks.json`».
3. Дальше обновляй файл по мере работы (правила ниже).

Схема `.claude/tasks.json`:

```json
{
  "project": "имя-папки-проекта",
  "updated": "2026-07-21T14:32:00Z",
  "tasks": [
    {
      "id": "t1",
      "title": "Короткий заголовок задачи",
      "detail": "1–2 предложения: что именно сделал / изменил.",
      "status": "done",
      "ts": "2026-07-21T14:30:00Z",
      "files": ["src/server.js", "index.html"]
    }
  ]
}
```

Поля:
- `updated` — ISO-время последнего изменения файла; обновляй при каждой записи.
- `tasks` — массив в хронологическом порядке, новые добавляй в конец.
  - `id` — последовательный: `t1`, `t2`, `t3`…
  - `title` — одна строка, на русском.
  - `status` — одно из: `todo`, `in_progress`, `done`.
  - `ts` — ISO-время; бери текущую дату из контекста сессии (точность до минут).
  - `detail`, `files` — по возможности, необязательно.

Файл всегда должен оставаться валидным JSON.

Когда и как записывать:
- Заводи запись на осмысленную единицу работы (фича, багфикс, рефактор, настройка) —
  то, что полезно видеть в списке «что сделано». Не логируй каждый вызов инструмента
  и мелкие промежуточные шаги.
- Многошаговую задачу добавь со статусом `in_progress`, по завершении переведи в
  `done` (обновляй ту же запись по `id`, не дублируй).
- Читай текущий файл, добавляй/меняй записи, пиши файл целиком обратно — так проще
  держать валидный JSON.
- Прошлые записи задним числом не переписывай (только смена статуса).
- Не клади в журнал секреты, токены, пароли, ключи.
